## 1 常用命令

### 1.1 镜像命令
```shell
#  查看镜像
docker images 

# 搜许镜像
docker search 镜像名称 
# NAME 名称
# DESCRIPTION 描述
# STARS  用户评价
# OFFICIAL 是否官方
# AUTOMATED 自动构建

# 拉取镜像 不写tag 会拉去latest版本 
docker pull 镜像名称:tag 

# 删除镜像
docker rmi 镜像mingc / 镜像id

#删除所有镜像 当运行的容器用到该镜像时 需停掉容器在删除镜像
docker rmi `docker images -q`
``` 


### 1.2 容器命令
##### 1.2.1 容器创建与运行
```shell
# 查看所有容器 运行中
docker ps

# 查看所有容器 
docker ps -a

# 查看最后一次运行的容器 
docker ps -l

# 查看停止运行的容器
docker ps -f status=exied

# 创建与启动容器
# 创建容器 
docker run
# -i 表示运行容器
# -t 表示容器启动后会进入命令行。加入这两个参数，容器创建并登陆进去，分配一个伪终端
# --name 容器名字
# -v 目录映射关系
# -d 会创建一个守护式容器在后台运行，（这样创建后不会自动登录容器）
# -p 表示端口映射，前者是宿主机端口，后者是容器内的映射端口，可以使用多个p做多个映射端口

#交互式创建容器
docker run -it -name=名称 镜像名称 命令

#守护式创建容器
docker run -di -name=名称 镜像名称 

#退出当前容器
exit

#登陆容器
docker run -it  镜像名称 命令

# 启动容器
docker start 容器名字/容器id
 
# 停止容器
docker stop 容器名字/容器id

# 重启容器
docker restart 容器名字/容器id
```

##### 1.2.2 容器 文件拷贝
```shell
#将文件拷贝到容器
docker cp 需要拷贝的文件或目录 容器名称:容器目录

# 将文件从文件拷贝出来
docker cp  容器名称:容器目录 需要拷贝的文件或目录

```

#### 1.2.3 目录挂载
```shell
 
# -v  宿主机目录:容器目录
docker run -di --name=mycentos5 -v D:/myhtml:/usr/local/myhtml centos:7
```
#### 1.2.4 查看容器的IP地址
```shell
# 查看容器运行的各种数据
docker inspect 容器名 / 容器id

#查看容器运行的IP
docker inspect --format='{{.NetworkSettings.IPAddress}}' mycentos5

```


#### 1.2.5 删除容器
```shell
# 删除容器 不能删除运行中的容器
docker rm 容器名 / 容器id
```
## 2.应用部署

### 2.1 mysql部署

```shell
# 拉取镜像 
docker pull mysql:5.7
# 创建容器
docker run -di --name=mysql_test -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql:5.7

# 查看容器  
docker ps 
```

### 2.2 nginx部署

```shell
# 拉取镜像 
docker pull nginx
# 创建容器
docker run -di --name=nginx_test -p 80:80 nginx
# 查看容器  
docker ps 
```

### 2.3 redis部署

```shell
# 拉取镜像 
docker pull redis
# 创建容器
docker run -di --name=redis_test -p 6379:6379 redis
# 查看容器  
docker ps 

# 进入容器
docker exec -it nginx_test /bin/bash

cd /etc/nginx
cat nginx.conf
cd /etc/nginx/conf.d
cd   /usr/share/nginx/html

#宿主
docker cp index.html nginx_test:usr/share/nginx/html
```

## 3.迁移与备份

```shell
#容器保存为镜像
docker commit 容器名称 镜像名称
 
docker commit nginx_test mynginx_i

# 镜像备份
docker save -o 镜像输出文件 镜像名称

docker save -o mynginx.tar mynginx_i
# 镜像恢复与迁移
docker load -i  镜像输出文件

docker load -i mynginx.tar
```

## 5.Docker私有仓库
```shell
# 拉取私有仓库镜像
docker pull registry
# 创建容器
docker run -di --name=registry -p 5000:5000 registry
#访问
http://localhost:5000/v2/_catalog
#访问镜像的标签列表
http://localhost:5000/v2/registry/tags/list
# 配置 
 "insecure-registries":["192.168.255.246:5000"]
 #linux
 vi /ect/docker/daemon.
 
# 私有仓库镜像上传

docker tag 镜像名称 私有仓库地址/镜像名称

docker tag mynginx_i 192.168.255.246:5000/mynginx_i

# 上传镜像
docker push 192.168.255.246:5000/mynginx_i

# 拉取镜像 
docker pull  192.168.255.246:5000/mynginx_i
```
